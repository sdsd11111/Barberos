# Objetivo

Definir cómo se siente usar BarberOS, para dos audiencias distintas: el dueño de la barbería (panel) y el cliente final (WhatsApp).

---

# Principio rector

El dueño de barbería no quiere aprender un software. La promesa central del producto es: *"No necesitas usuario ni contraseña, ni aprender un dashboard."* Toda decisión de UX se mide contra esa promesa.

---

# Experiencia del cliente final (vía WhatsApp)

No existe app que descargar. No existe login. El flujo completo:

1. El cliente escanea un QR o recibe el link directamente.
2. Recibe un mensaje de WhatsApp con su progreso visual: `[███░░] 3 de 5`.
3. Al llegar al quinto corte, recibe el mensaje de premio.
4. Se le pide una calificación simple (1-5) por el mismo canal.

No hay pantallas. No hay fricción. El canal es uno que el cliente ya usa a diario.

---

# Experiencia del dueño (Panel)

## Principio: "Sin dashboard que aprender"

El panel existe para dos acciones diarias, no para navegar reportes complejos:

1. Registrar un corte (modal simple: número de WhatsApp + barbero).
2. Aprobar o rechazar check-ins entrantes (cola de aprobación con polling en tiempo real).

Todo lo demás (clientes, historial, métricas) es secundario y de consulta, no de uso diario obligatorio.

## Cola de aprobación (Anti-fraude)

Cada check-in vía WhatsApp entra como `PENDING`. El dueño recibe una notificación push nativa en su celular (Sprint 8, 2026-07-21) con sonido, incluso si el panel está cerrado. Al abrir el panel, ve la cola visual tipo bandeja y aprueba o rechaza con un toque.

**Mecanismo real (confirmado por versión del hijo, 2026-07-22):** Push nativo con sonido como canal primario, cola visual como fallback cuando el panel está abierto. Detalle técnico completo de la especificación pendiente de confirmar con el hijo.

Este mecanismo evita que un cliente reporte visitas falsas y le da al dueño una sensación de control real sobre lo que se registra — refuerza directamente el eje emocional de "control", no solo la integridad técnica del dato.

## Identidad visual (cinematográfica)

- Fondo: negro cálido (`#0a0807`), no negro puro — transmite calidez, no frialdad de software corporativo.
- Texto: crema (`#f3ece1`).
- Acento: naranja cobrizo (`#d97644`) — cálido, artesanal, coherente con el oficio de barbería.
- Tipografías: Fraunces (display, con carácter editorial) + Space Grotesk (interfaz) + JetBrains Mono (datos, códigos, métricas).

Esta identidad diferencia a BarberOS de la estética típica de SaaS genérico (azul corporativo, sans-serif neutro) y refuerza el posicionamiento premium descrito en el roadmap comercial.

# Principios de interacción

1. **Cero curva de aprendizaje para el cliente final.** Si requiere explicación, está mal diseñado.
2. **Mínimas pantallas para el dueño.** Cada pantalla adicional es fricción que compite contra su tiempo cortando cabello.
3. **Feedback inmediato.** Toda acción (registrar corte, aprobar check-in) debe confirmar visualmente en menos de 2 segundos. El canal primario es push nativo con sonido (Sprint 8); el polling agresivo de versiones anteriores es ahora fallback, no el mecanismo principal.
4. **El progreso siempre visible.** La barra de progreso Unicode es el elemento central de motivación del cliente final — no debe reemplazarse por abstracciones (puntos, niveles) que pierdan la simpleza visual.

---

# Lo que se evita deliberadamente

- Onboarding con tours guiados o tooltips excesivos — contradice la promesa de simplicidad.
- Cualquier pantalla que obligue al dueño a "configurar" antes de ver valor. El primer corte registrado debe generar valor visible de inmediato.
- Terminología técnica visible al usuario (API, webhook, tenant) — eso vive solo en la documentación interna.

---

# Relación con el Motor de Conocimiento e IA (Premium)

Cuando se libere el Director IA (documento 08), la interacción debe mantenerse dentro del mismo canal de WhatsApp o, como máximo, un chat simple dentro del panel — nunca un dashboard de reportes que el dueño deba interpretar por sí mismo. La IA existe para traducir datos en lenguaje conversacional, no para agregar una pantalla más que aprender.
